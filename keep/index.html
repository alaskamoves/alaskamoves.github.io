In-House Bookkeeping Interface

The keep/ directory now serves as the centralized bookkeeping interface, linking expense and mileage trackers through the date spine yyyymmdd to ensure on-premise LLC compliance.

This section interacts directly with the underlying SQLite database (ledger.db), supports manual editing via DataGrip, and can be integrated with automation scripts for streamlined data ingestion and management using canonical field names.

The schema uses canonical, lower_snake_case field names for each table:

  • Transactions (table: txns): yyyymmdd, house, type, amount, memo
  • Mileage (table: miles): yyyymmdd, business_miles, note, optional odometer_start, odometer_end
  • Posts (tables: posts, post_mirrors): post_id, yyyymmdd, title, canonical_url; and mirrors: platform, url

The schema enforces data integrity with explicit constraints: yyyymmdd is 8 digits; amount and business_miles are numeric; post_id and house are non-empty; and foreign keys link post_mirrors.post_id to posts.post_id.
